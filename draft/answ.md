1. ¿Cuáles son las “Cuatro Libertades” definidas por la Free Software Foundation?

Las Cuatro Libertades** definidas por la Free Software Foundation (FSF) son los principios fundamentales que garantizan que un software sea considerado libre. Estas libertades son:

1. Libertad 0: La libertad de usar el programa para cualquier propósito.  
   - Esto significa que no hay restricciones en cómo se puede utilizar el software, ya sea para uso personal, comercial, educativo, etc.

2. Libertad 1: La libertad de estudiar cómo funciona el programa y adaptarlo a tus necesidades.  
   - Esto implica tener acceso al código fuente del software, lo cual es esencial para poder entenderlo y modificarlo.

3. Libertad 2: La libertad de distribuir copias del programa para ayudar a otros.  
   - Esto permite compartir el software con otras personas, ya sea de forma gratuita o cobrando por la distribución.

4. Libertad 3: La libertad de mejorar el programa y hacer públicas esas mejoras para beneficio de toda la comunidad.  
   - Esto significa que puedes modificar el software y distribuir tus versiones modificadas, siempre y cuando también sean libres.

Estas libertades aseguran que los usuarios tengan control sobre el software que utilizan y promueven la colaboración y el bien común.


2. ¿Cómo justifica el Manifiesto GNU la importancia ética del software libre?

El Manifiesto GNU, escrito por **Richard Stallman, justifica la importancia ética del software libre desde varios ángulos:

1. Cooperación y solidaridad:  
   - Stallman argumenta que compartir software es una forma de cooperación entre personas. Restringir el acceso al software va en contra de la solidaridad humana y la ayuda mutua.

2. **Libertad del usuario**:  
   - El software libre respeta la libertad de los usuarios para usar, estudiar, modificar y distribuir programas. Stallman considera que estas libertades son esenciales para una sociedad justa y equitativa.

3. **Evitar la dependencia de sistemas privativos**:  
   - El software privativo (no libre) crea dependencia de los usuarios hacia las empresas que lo controlan. Stallman ve esto como una forma de opresión, ya que los usuarios no tienen control sobre las herramientas que utilizan.

4. **El "Golden Rule" (Regla de Oro)**:  
   - Stallman utiliza la **Regla de Oro** ("trata a los demás como quieres que te traten") para justificar la importancia de compartir software. Si te gusta un programa, es ético compartirlo con otros que también lo necesiten.

5. **Beneficio para la sociedad**:  
   - El software libre fomenta la innovación y el progreso tecnológico al permitir que cualquiera pueda contribuir y mejorar los programas. Esto beneficia a toda la sociedad, no solo a unos pocos.

En resumen, el Manifiesto GNU defiende que el software libre es una cuestión de **ética**, **libertad** y **justicia social**, no solo una cuestión técnica o económica.

### 3. **¿Cuál es la diferencia entre "código abierto" y "software libre"?**

Aunque los términos **"código abierto"** y **"software libre"** a menudo se usan indistintamente, tienen diferencias importantes en su enfoque y filosofía:

1. **Enfoque filosófico**:
   - **Software libre**:  
     - Se centra en las **libertades del usuario** (las Cuatro Libertades).  
     - Tiene una base ética y moral: el software debe respetar la libertad de los usuarios.  
     - Promovido por la **Free Software Foundation (FSF)** y Richard Stallman.
   - **Código abierto**:  
     - Se centra en los **beneficios prácticos** de compartir el código fuente, como la colaboración y la mejora del software.  
     - Tiene un enfoque más técnico y empresarial.  
     - Promovido por la **Open Source Initiative (OSI)**.

2. **Términos y definiciones**:
   - **Software libre**:  
     - Define el software en términos de **libertades** (las Cuatro Libertades).  
     - No necesariamente implica que el software sea gratuito (puede venderse), pero sí que respete las libertades del usuario.
   - **Código abierto**:  
     - Define el software en términos de **acceso al código fuente** y las condiciones bajo las cuales se puede modificar y redistribuir.  
     - No siempre garantiza todas las libertades del software libre.

3. Licencias:
   - Software libre:  
     - Utiliza licencias como la GNU GPL (General Public License), que garantiza que el software y sus derivados sigan siendo libres.
   - Código abierto:  
     - Utiliza licencias como la MIT License o la Apache License, que permiten el uso del código fuente pero no siempre garantizan que las modificaciones sean libres.

4. Ejemplo práctico:
   - Un programa puede ser código abierto pero no software libre si, por ejemplo, permite el acceso al código fuente pero restringe su uso comercial o la redistribución de versiones modificadas.
